1.私有变量用**双下划线__+变量名**命名

2.**super()init_()** 只会调用父类的 **init 方法（构造方法）**，且仅执行该方法内部的代码逻辑，不会 “顺带” 调用父类的其他方法 / 属性

4.当对被 **@property 装饰的属性**做「赋值操作（=）**时，Python 会自动触发对应的 @xxx.setter装饰方法**；而self.area里的area继承自父类 Shape，因此触发的是父类的@area.setter 方法

5.子类和父类有相同的方法时，遵循**先子类后父类**的原则

6.类变量的初始化代码（`total_book_types = 0`、`total_inventory = 0`）**不是「第一次调用类」执行**，而是**在 Python 解释器加载并定义这个 `Book` 类的那一刻，就会执行且仅执行一次**，后续无论创建多少个 `Book` 实例、甚至继承这个类，初始化代码都不会再执行。

7.执行**raise 自定义异常类**的全过程：

     1）.触发raise语句，中断函数执行

     2）.创建异常对象

     3）.异常向上传播（寻找 except 捕获），如果找不到匹配的except，程序会发生崩溃

8.使用setter方法的前提：对私有变量进行赋值操作

9.`InventoryError(f"未找到ISBN为 {isbn} 的图书")` 会：

- 调用 `InventoryError` 的构造方法（继承自`Exception`的`__init__`）；
- 将格式化字符串作为参数传入，存储到异常对象的`args`属性中（这是 Python 异常的默认行为）；
- 生成一个包含「错误提示 + 调用栈」的异常对象。
